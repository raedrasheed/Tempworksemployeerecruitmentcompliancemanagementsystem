## 3. Interpretation

- **`mismatch=0` everywhere** ‚Äî the backfill never assigned a child a
  tenantId that disagrees with its parent. Confidence in the chain is
  total for the rows it did fill.
- **`notifications.withoutTid = withoutParentTid = 2`** ‚Äî two rows have
  `tenantId=NULL`, but the parent user's `agencyId` is also NULL. That
  is a legitimate NULL (platform-admin). `unexplainedNulls = 0`.
- **`unexplainedNulls = 0` across all populated tables** ‚Äî there is no
  child whose parent has a tenantId but the child does not. Every row
  the engine could classify is classified.

## 4. Acceptance gate

For staging promotion of a flag profile that includes
`TENANT_SAFE_REPORTS_ENABLED=true`, the verifier must report:

- `mismatch = 0` across all models present
- `unexplainedNulls = 0` across all models present

Both conditions are satisfied on the current fixture.

## 5. Reproducing

```sh
cd backend
DATABASE_URL=... npm run saas:phase2-verify-entity-tenantids
```

Exit code:
- `0` ‚Äî all models passed (or were ERROR/skipped)
- `2` ‚Äî at least one model had `mismatch>0` or `unexplainedNulls>0`
- `3` ‚Äî runtime error
